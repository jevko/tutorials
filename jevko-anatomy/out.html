Anatomy of a jevko

This is an informal description of the various elements of the Jevko syntax.

The name Jevko /ˈdʒef.kɔ/ is derived from Polish drzewko /ˈdʐɛf.kɔ/, meaning small tree.

This relates to the fact that Jevko is meant to be a minimal syntax for simple flexible tree structures.

We shall call a specific instance of a Jevko tree a jevko.

An example of a jevko looks like this:

first name [string]
last name [string]
is alive [boolean]
age [integer]
address [
  street address [string]
  city [string]
  state [string]
  postal code [string]
]
children [list]
spouse []
object

We will now break this example down and identify and name its parts.

top jevko

Notice that the jevko is not wrapped in square brackets. A jevko which is not wrapped in square brackets is the top or top-level jevko.

parts of jevkos

A jevko is made out of a number of subjevkos followed by a suffix:

first name [string]last name [string]is alive [boolean]age [number]address [
  street address [string]
  city [string]
  state [string]
  postal code [string]
]children [array]spouse []object

subjevkos and their parts

A subjevko is a kind of a name-value pair:

first name [string]

A subjevko is made out of a prefix followed by a jevko wrapped in [square brackets]:

first name [string]

nested jevkos

A jevko which is wrapped in square brackets is called a nested jevko.

trivial jevkos and subjevkos

Notice that the jevko nested in the above subjevko has no subjevkos of its own, only a suffix:

string

A jevko without subjevkos like this is called trivial.

A subjevko whose jevko is trivial is itself called trivial.

empty jevkos

Now let's look at this subjevko:

spouse []

The jevko in this subjevko can't be seen -- it has no subjevkos and an empty suffix.

Such a jevko is called empty.

An empty jevko is a special case of a trivial jevko.

complex jevkos and subjevkos

Another interesting subjevko is this one:

address [  street address [string]
  city [string]
  state [string]
  postal code [string]
]

Its nested jevko has 4 subjevkos:

  street address [string]  city [string]  state [string]  postal code [string]

A jevko with at least one subjevko is called complex.

A subjevko whose jevko is complex is itself called complex.

Whitespace

Whitespace is characters such as spaces, new lines, and tabs.

text

All whitespace in a jevko is part of a text.

A text is a prefix or a suffix.

blank text

Text which is made only of whitespace is called blank. Blank text may be empty.

blank jevkos

A trivial jevko with a blank suffix is itself called blank. For example:

blank subjevkos

A subjevko with a blank prefix and a blank jevko is called blank. For example:

  [  ]

empty subjevkos

A subjevko with an empty prefix and an empty jevko is called empty. It looks like this:

[]

new lines

Notice that in a jevko like this:

first name [string]last name [string]object

The new line after the first subjevko is actually part of the second subjevko's prefix (shown below as ¶ for clarity):

first name [string]¶
last name [string]

Similarly, the new line after the second subjevko is part of the jevko's suffix:

last name [string]¶
object

Similarly in this subjevko's jevko:

address [  street address [string]
  city [string]
  state [string]
  postal code [string]
]

the new line after the opening bracket [ is part of the prefix of the first subjevko:

address [¶
  street address [string]  city [string]  state [string]  postal code [string]]

Also the new line before the closing bracket ] is the suffix of the jevko:

address [  street address [string]  city [string]  state [string]  postal code [string]¶
]

Similar thing is true for all prefixes and suffixes. All whitespace is captured by them.
